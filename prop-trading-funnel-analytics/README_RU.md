# prop-trading-funnel-analytics

Портфолио-кейс по аналитике воронки челленджевой проп-трейдинговой платформы. Проект анализирует полный путь трейдера — от регистрации до выплаты — и явно рассматривает качество трейдеров и экспозицию выплат как ключевые бизнес-ограничения, а не второстепенные параметры.

Репозиторий построен как воспроизводимый аналитический проект:

- синтетическая генерация данных на Python
- аналитический SQL на DuckDB
- генерация графиков на Python
- облегчённый notebook-walkthrough
- краткий гайд для интервью
- сохранённые артефакты для просмотра без запуска пайплайна

*English version: [README.md](README.md)*

## Бизнес-контекст

Проп-трейдинговые платформы не побеждают за счёт максимизации регистраций или первых покупок. Когорта с высокой конверсией на верхних этапах воронки всё равно может разрушить экономику платформы, если она:

- проходит оценку с высоким процентом успеха
- быстро достигает финансирования
- генерирует крупные одобренные или ожидающие выплаты относительно выручки от сборов

Операционный вопрос — не просто «кто конвертируется?», а «какие пользователи создают ценность, какие создают обязательства, и где воронка перестаёт быть привлекательной для бизнеса?»

## Постановка задачи

Анализ синтетического датасета 2025 года для челленджевой проп-трейдинговой платформы по воронке:

`регистрация → KYC → покупка челленджа → фаза 1 → фаза 2 → финансирование → выплата`

Ключевые вопросы:

1. Где самые большие разрывы в воронке?
2. Какие каналы привлечения приводят лучших трейдеров, а не просто больше регистраций?
3. Какие типы челленджей балансируют конверсию с качеством downstream?
4. Какие страны и сегменты по опыту хорошо конвертируются в финансированных трейдеров?
5. Какие финансированные сегменты генерируют наибольшую экспозицию выплат?
6. Какие поведенческие паттерны трейдеров связаны с продвижением или провалом?
7. Какие сегменты выглядят привлекательно на верхних этапах воронки, но слабо — с учётом экспозиции выплат?

## Снимок данных

Сгенерированная выборка:

- 18 000 регистраций
- 8 566 верифицированных по KYC пользователей
- 2 232 покупателя
- 2 676 покупок челленджей
- 580 финансированных челленджей
- 302 одобренных выплаты

Воронка верхнего уровня:

- Начало KYC: 60,5% от регистраций
- Верификация KYC: 47,6% от регистраций
- Конверсия верифицированных в покупку: 26,1%
- Процент прохождения фазы 1: 47,9% от купленных челленджей
- Процент прохождения фазы 2: 54,2% от прошедших фазу 1
- Процент одобренных выплат: 52,1% от финансированных челленджей

## Схема данных

### `users.csv`

Одна строка на регистрацию.

| Колонка | Описание |
| --- | --- |
| `user_id` | Уникальный идентификатор пользователя |
| `registration_date` | Дата регистрации |
| `country` | Страна пользователя |
| `acquisition_channel` | `organic`, `paid_social`, `affiliates`, `influencers`, `direct`, `community` |
| `device_type` | Desktop, mobile или tablet |
| `platform_language` | Язык платформы при регистрации |
| `age_bucket` | Возрастная группа |
| `prior_trading_experience` | `none`, `beginner`, `intermediate`, `advanced` |

### `kyc_events.csv`

Одна строка на статус KYC пользователя.

| Колонка | Описание |
| --- | --- |

| `kyc_started_at` | Время начала KYC |
| `kyc_completed_at` | Время завершения KYC |
| `kyc_status` | `verified`, `rejected`, `abandoned`, `pending`, `not_started` |

### `challenges.csv`

Одна строка на купленный челлендж.

| Колонка | Описание |
| --- | --- |
| `challenge_id` | Идентификатор челленджа |
| `user_id` | Пользователь-покупатель |

| `challenge_type` | `standard`, `aggressive`, `swing`, `low_cost_trial` |
| `price_usd` | Стоимость челленджа |
| `account_size` | Размер симулируемого счёта |
| `promo_code_used` | Промокод или null |

### `challenge_progress.csv`

Одна строка на статус прохождения челленджа.

| Колонка | Описание |
| --- | --- |
| `challenge_id` | Идентификатор челленджа |
| `phase_1_status` | Результат фазы 1 |
| `phase_1_completed_at` | Временная метка завершения фазы 1 |

Одна строка на запрос выплаты.

| Колонка | Описание |

### `trader_behavior.csv`

Одна строка на покупателя.

| Колонка | Описание |
| --- | --- |
| `user_id` | Идентификатор пользователя |

| `avg_rr` | Среднее соотношение риск/доходность |

| `inactivity_days_after_purchase` | Прокси неактивности после покупки |

## Определения метрик

- `CR` — конверсия с одного этапа на следующий.
- `Funded conversion` — доля купленных челленджей, достигших статуса финансирования.
- `Payout rate` — доля финансированных челленджей с хотя бы одной одобренной выплатой.
- `Payout exposure` — одобренные и находящиеся на рассмотрении суммы выплат, используемые как прямой прокси обязательств.
- `Churn / inactivity proxy` — `inactivity_days_after_purchase`, синтетическая мера незаинтересованности после покупки.

## Методология

### 1. Синтетическая генерация данных

- Регистрации по реалистичным странам, каналам, возрастам, устройствам, языкам и уровням опыта.
- Симуляция прохождения KYC с учётом канала, устройства, соответствия требованиям и вовлечённости.
- Симуляция покупок челленджей с повторными попытками, выбором типа и использованием промокодов.
- Поведенческие метрики трейдеров, привязанные к навыкам, дисциплине, вовлечённости и риску.
- Симуляция прохождения фаз 1, 2, финансирования и выплат.

### 2. SQL-аналитика

Папка `sql/` содержит семь production-style запросов:

- `01_overall_funnel.sql`
- `02_funnel_by_acquisition_channel.sql`
- `03_funnel_by_challenge_type.sql`
- `04_funded_payout_analysis.sql`
- `05_cohort_analysis_by_registration_month.sql`
- `06_trader_segment_comparison.sql`
- `07_stage_timing.sql`

Запросы используют CTE и понятное именование для построения воспроизводимых аналитических таблиц в DuckDB.

### 3. Python-отчётность

Python в `src/analysis.py` читает сгенерированные данные и сохраняет семь графиков в `outputs/`:

- диаграмма этапов воронки
- конверсия по каналу привлечения
- конверсия по типу челленджа
- тепловая карта прогрессии когорт
- экспозиция выплат по сегменту
- поведение трейдера vs исход воронки
- время прохождения этапов (медиана дней между переходами)

## Ключевые выводы

### 1. Воронка ломается сильнее всего на KYC и монетизации после KYC

- Только **60,5%** регистраций начинают KYC.
- Только **47,6%** регистраций проходят верификацию KYC.
- Только **26,1%** верифицированных пользователей покупают челлендж.

Интерпретация: крупнейший коммерческий разрыв не в сложности оценки. Платформа теряет большую долю пользователей ещё до начала монетизации.

### 2. Каналы, приводящие лучших трейдеров, — не те, что дают лучшую юнит-экономику

Сравнение каналов:

- `direct`: конверсия регистрация→покупка **20,7%**, покупка→финансирование **32,0%**
- `community`: **18,1%** и **33,0%** соответственно
- Оба значительно сильнее `paid_social` (10,9%) и `influencers` (4,8%) по качеству трейдеров

Но юнит-экономика переворачивается downstream:

- `community` даёт gross profit proxy **−79,0k$**
- `direct` — **−26,0k$**
- `paid_social` остаётся положительным, потому что слабое downstream-качество сдерживает обязательства по выплатам

Каналы, которые стоит защищать ради качества трейдеров, — именно те, что создают наибольшие обязательства по выплатам. Оптимизация по регистрациям или конверсии в funded ведёт в неправильную сторону.

![Конверсия по каналам](outputs/conversion_by_acquisition_channel.png)

### 3. Low-cost trials помогают привлечению, но не являются качественным продуктом

- `low_cost_trial` генерирует **693** челленджа
- Финансирует лишь **8,1%** купленных челленджей
- Trials дают объём и выручку, но их downstream-показатели настолько слабы, что их следует отслеживать отдельно от основного продукта

### 4. Swing-счета создают наиболее очевидный компромисс по обязательствам

- `swing` имеет наивысшую конверсию в финансирование — **35,7%**
- Но и наихудший gross profit proxy — **−66,9k$**
- Соотношение экспозиции выплат к сборам — **1,24x** (обязательства превышают выручку)

Напротив, `aggressive` — сильнейший продукт по gross profit proxy (**+82,5k$**), но с высоким funded-to-payout rate 52,9%.

### 5. Опыт торговли сильно влияет на поздних этапах

Конверсия в финансирование по опыту:

- `advanced`: **44,2%**
- `intermediate`: **25,5%**
- `beginner`: **6,3%**
- `none`: **1,4%**

Уровень опыта — предиктор и роста, и риска обязательств.

### 6. Результаты по странам неравномерны и намеренно зашумлены

Среди стран с не менее чем 70 челленджами наилучшая конверсия в финансирование — в Польше, Франции, ОАЭ, Испании, Германии, Канаде. Сигнал следует рассматривать как направленный, а не как политику — эффекты стран в реальных данных смешивают регулирование, платёжные барьеры и состав каналов.

### 7. Поведение разделяет прогрессирующих и ранних провалов

Средние показатели по исходу:

- Одобренная выплата: **0,60%** риска на сделку, **50%** win rate, **1,29** RR, **0,50** нарушений
- Провал фазы 1: **0,89%** риска, **42%** win rate, **1,04** RR, **1,15** нарушений

Разделение между группами видно до стадии финансирования — значит, команды по риску и поддержке могут действовать превентивно.

![Поведение vs исход](outputs/behavior_vs_funnel_outcomes.png)

### 8. Некоторые комбинации сегментов вводят в заблуждение на верхних этапах воронки

Примеры: `direct | advanced`, `community | advanced`, `organic | advanced`. Эти сегменты отлично конвертируются и финансируются, но создают такую экспозицию выплат, что становятся непривлекательными по gross profit proxy.

### 9. Время прохождения этапов показывает, где накапливается трение

Медианы в днях (полный IQR — в `outputs/stage_timing.csv`):

- Регистрация → KYC: широкий IQR указывает на трение в самом KYC-процессе
- KYC → первая покупка: P75 заметно выше медианы — часть верифицированных пользователей откладывает покупку на недели
- Покупка → исход фазы 1: провалы происходят быстрее успехов (нарушение лимита просадки на раннем этапе)
- Финансирование → первый запрос выплаты: IQR здесь важнее всего для финансового планирования

## Revenue Impact — Влияние на выручку

**Конверсия KYC→покупка (26,1% верифицированных)**

Это наибольший адресуемый разрыв без дополнительного маркетингового бюджета. Каждый процентный пункт улучшения конверсии даёт ~86 дополнительных покупок. При средней стоимости челленджа ~$250 — это около **$21 500 дополнительной выручки** при нулевом дополнительном CAC.

Улучшение на 3 п.п. (26% → 29%) транслируется в ~$64 500 инкрементной выручки из существующей базы верифицированных пользователей. Прежде чем действовать: выяснить, объясняется ли отток ценовой чувствительностью, UX-трением или изначальным отсутствием намерения покупать.

**Начало KYC (60,5% регистраций)**

39,5% регистраций не начинают KYC. Улучшение показателя начала с 60,5% до 65% добавляет ~810 пользователей, начавших KYC. При текущем уровне верификации (~78,7%) — ~638 дополнительных верифицированных, ~166 дополнительных покупателей, ~$41 500 выручки.

**Важно:** это приблизительные оценки порядка величины. Реальный эффект зависит от того, является ли отток однородным по каналам — он таковым не является.

## Бизнес-рекомендации

### Рост

- Не оптимизировать привлечение по регистрациям или первым покупкам.
- Рассматривать `organic` как наиболее сбалансированный канал масштабирования: приемлемое качество при положительном gross profit proxy.
- Держать `low_cost_trial` как продукт для лидогенерации, отделив от отчётности по качеству трейдеров.

### Риск

- Усиленный мониторинг финансированной книги для `community | advanced` и `direct | advanced`.
- Использовать экспозицию выплат на финансированный челлендж как постоянный KPI риска рядом с payout rate.
- Просматривать долю выплат в статусе "под рассмотрением" для продвинутых платных и партнёрских когорт перед масштабированием.

### Продукт

- Пересмотреть цены или ужесточить условия `swing`, если он остаётся структурно отрицательным.
- Рассмотреть дифференцированные политики прогрессии или выплат по типу челленджа.

## Ограничения

- Синтетический датасет. Взаимосвязи разработаны как реалистичные, а не измерены на реальных данных.
- Бизнес-прокси не учитывает CAC, расходы на поддержку, чарджбэки, хеджирование и постоянные накладные расходы.
- Результаты по странам чувствительны к смоделированному составу канала и опыта.
